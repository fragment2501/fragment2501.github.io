--- layout: post title: Fixing the SID of a copied virtual machine date: '2009-06-10T08:11:00.000-07:00' author: Bryan tags: - VMWare modified_time: '2009-06-11T15:39:45.366-07:00' blogger_id: tag:blogger.com,1999:blog-19111454.post-5526946864010491212 blogger_orig_url: http://www.bryansgeekspeak.com/2009/06/fixing-sid-of-copied-virtual-machine.html --- I use VMWare virtuals for all the new servers I build at work, and the other day I ended up building a sandbox domain using copies of base windows 2003 virtual I'd built a few months ago. With one domain controller in place, I joined another VM to the domain, which seemed to work just fine. After a reboot, I tried to login to the domain from the new server and got this error:

"The name or security ID (SID) of the domain specified is inconsistent with
the trust information for that domain"

After a bit of digging on google, I found that its an issue with one or more servers on the same domain using the same SID. Since both my servers in this case were copies of the same base windows 2003 server, I was running into exactly that issue. I was able to fix the problem by changing the SID of my new server using the NewSID tool.

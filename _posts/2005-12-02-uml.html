--- layout: post title: UML date: '2005-12-02T12:47:00.000-08:00' author: Bryan tags: modified_time: '2005-12-16T07:52:16.546-08:00' blogger_id: tag:blogger.com,1999:blog-19111454.post-113355651045632801 blogger_orig_url: http://www.bryansgeekspeak.com/2005/12/uml.html --- I've started working on UML diagrams in Java class and I'm now building UML diagrams at work for documentation. I've been using this list for UML tools and info:

http://plg.uwaterloo.ca/~might/uml.html

For Java, I see where using a tool like UMLet works great. At work I'm documenting a coldfusion MX project where all the documentation is currently in MS Word. To make the UML diagram useful, I need to be able to embed the diagram into the existing documentation. I've moved from UMLet to Visio so that I can OLE link diagrams directly in Word. The linking part is great, but doing UML in visio is quite painful. UMLet allows you to simply type your attributes and methods into an editor window and the diagram updates as you type, making data entry easy and fast. Visio makes the process much more painful...to edit a class, you first open its properties window, select either "attributes" or "operations", then select individual attributes to edit them. Even worse, visio forces you to select the permissions and datatypes from a dropdown, adding clicks to data entry. Gah! What a pain.

Perhaps after I'm finished with this set of documentation I'll take some time to look at more portable options like OpenOffice and see if they have better tools for the job.
